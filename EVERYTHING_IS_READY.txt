❓ ANSWERS TO CLIENT QUESTIONS
════════════════════════════════════════════════════════════════════════════

Q: How do I use this?
A: 3 ways:
   1. Manual: POST /sync/all anytime
   2. Automatic: Runs daily at midnight UTC
   3. Monitor: GET /status to see metrics

Q: What if it fails?
A: Automatic retry logic (3 attempts with backoff). Errors logged.
   Status endpoint shows last error.

Q: Duplicates?
A: Zero. Composite key (source + externalJobId) prevents them.
   We tested it twice - second run inserted 0 new jobs.

Q: How many jobs?
A: ~700 per run (Adzuna 150, USAJobs 500, Greenhouse 45)

Q: How fast?
A: 10-12 seconds for complete sync of all 3 sources

Q: What do I need to do?
A: Nothing! It runs automatically. Or call POST /sync/all manually.

Q: How do I query jobs?
A: Directly via MongoDB. Search API can be added in Phase 2.

Q: Cost?
A: Zero incremental cost. API credits were pre-configured.
